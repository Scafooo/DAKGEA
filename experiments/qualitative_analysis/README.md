Human: # Qualitative Analysis Framework for DAKGEA

A comprehensive toolkit for analyzing the quality of synthetic entities generated by PLM augmentation.

## 📦 Features

### 1. **Diversity Metrics**
Measures how varied and non-redundant the generated entities are:
- **Attribute Value Diversity**: Unique values per attribute
- **Lexical Diversity (TTR)**: Type-Token Ratio for vocabulary richness
- **Embedding Diversity**: Semantic distance in embedding space
- **Self-BLEU**: N-gram overlap (lower = more diverse)
- **Structural Diversity**: Variance in entity structure

### 2. **Realism Metrics**
Evaluates how plausible and well-formed the generated entities are:
- **Attribute Validity**: Date/number formatting, no empty values
- **Fluency**: Text quality (grammar, capitalization)
- **Semantic Coherence**: Do attributes make sense together?
- **Alignment Consistency**: Are aligned pairs semantically related?

### 3. **Sample Extraction**
Extracts representative samples for human evaluation:
- **Random Sampling**: Unbiased random selection
- **Diverse Sampling**: Maximizes diversity using embeddings
- **Aligned Sampling**: Focuses on entities with aligned pairs
- **TSV Export**: Ready for spreadsheet annotation

### 4. **Quality Reports**
Generates comprehensive analysis reports:
- **JSON Format**: Machine-readable metrics
- **Markdown Format**: Human-readable with interpretations
- **Automated Recommendations**: Suggests parameter adjustments
- **Sample Extraction**: Includes samples for manual review

## 🚀 Quick Start

### Generate Complete Quality Report

```bash
python -m experiments.qualitative_analysis.quality_report \
    --original data/raw/openea/BBC_DB \
    --augmented results/BBC_DB_01_05/augmentation \
    --output-dir results/quality_analysis \
    --n-samples 50
```

This creates:
- `quality_report.md` - Human-readable report
- `quality_report.json` - Machine-readable metrics
- `samples/random_samples.tsv` - Random samples for annotation
- `samples/diverse_samples.tsv` - Diverse samples for annotation

### Python API Usage

```python

report = generate_quality_report(
    original_path="data/raw/openea/BBC_DB",

print(f"Novelty: {report['diversity']['novelty_ratio']:.2%}")
print(f"Fluency: {report['realism']['fluency_rate']:.2%}")
```

## 📈 Interpreting Metrics

### Diversity Metrics

| Metric | Target Range | Interpretation |
|--------|-------------|----------------|
| `novelty_ratio` | 0.4 - 0.7 | % of new values (not in originals) |
| `embedding_diversity_synthetic` | 0.3 - 0.6 | Semantic distance (higher = more diverse) |
| `self_bleu_synthetic` | < 0.3 | N-gram overlap (lower = less repetitive) |
| `lexical_diversity_synthetic` | 0.3 - 0.7 | Vocabulary richness |

### Realism Metrics

| Metric | Target | Interpretation |
|--------|--------|----------------|
| `fluency_rate` | > 0.8 | % of well-formed text |
| `empty_value_rate` | < 0.1 | % of missing values (should be low) |
| `date_validity_rate` | > 0.9 | % of properly formatted dates |
| `semantic_coherence_mean` | 0.3 - 0.6 | Attributes make sense together |
| `alignment_consistency_mean` | > 0.5 | Aligned pairs are semantically related |

## 🔧 Advanced Usage

### Custom Analysis

```python
from experiments.qualitative_analysis import DiversityAnalyzer, RealismAnalyzer
from src.core.data_io import load_dataset

# Load datasets
orig = load_dataset("data/raw/openea/BBC_DB")
aug = load_dataset("results/BBC_DB_01_05/augmentation")

# Diversity analysis
div_analyzer = DiversityAnalyzer(embedding_model="all-MiniLM-L6-v2")
div_metrics = div_analyzer.analyze(orig, aug, stage="augmentation")

# Realism analysis
real_analyzer = RealismAnalyzer(embedding_model="all-MiniLM-L6-v2")
real_metrics = real_analyzer.analyze(orig, aug, stage="augmentation")

# Print specific metrics
print(f"Novelty: {div_metrics['novelty_ratio']:.2%}")
print(f"Embedding Diversity: {div_metrics['embedding_diversity_synthetic']:.3f}")
print(f"Fluency: {real_metrics['fluency_rate']:.2%}")
```

### Custom Sampling Strategy

```python
from experiments.qualitative_analysis import EntitySampler

sampler = EntitySampler(seed=42)

# Extract diverse samples
samples = sampler.extract_samples(
    orig, aug,
    n_samples=100,
    strategy="diverse",  # Maximizes semantic diversity
    stage="augmentation"
)

# Export to TSV for annotation
sampler.export_to_tsv(
    samples,
    output_path="for_annotation.tsv",
    include_comparison=True  # Include closest original entity
)
```

## 📝 Human Evaluation Workflow

1. **Extract Samples**:
   ```bash
   python -m experiments.qualitative_analysis.entity_sampler \
       --original data/raw/openea/BBC_DB \
       --augmented results/BBC_DB_01_05/augmentation \
       --output samples_for_review.tsv \
       --n-samples 50 \
       --strategy diverse
   ```

2. **Annotate in Spreadsheet**:
   - Open `samples_for_review.tsv` in Excel/Google Sheets
   - Rate `realism_score` (1-5): How realistic is the entity?
   - Rate `consistency_score` (1-5): Are aligned pairs consistent?
   - Add `notes` about any issues

3. **Analyze Annotations**:
   ```python
   import pandas as pd

   df = pd.read_csv("samples_for_review.tsv", sep="\t")
   print(f"Avg Realism: {df['realism_score'].mean():.2f}")
   print(f"Avg Consistency: {df['consistency_score'].mean():.2f}")
   ```

### Hyperparameter Tuning

Compare quality across different augmentation parameters:

```python
import json
from pathlib import Path

experiments = ["BBC_DB_01_01", "BBC_DB_01_02", "BBC_DB_01_03"]
results = {}

for exp in experiments:
    report_path = Path(f"results/{exp}/quality_analysis/quality_report.json")
    with report_path.open() as f:
        report = json.load(f)

    results[exp] = {
        "novelty": report["diversity"]["novelty_ratio"],
        "fluency": report["realism"]["fluency_rate"],
        "consistency": report["realism"]["alignment_consistency_mean"],
    }

# Find best configuration
best = max(results.items(), key=lambda x: x[1]["fluency"])
print(f"Best experiment: {best[0]}")
```

### Quality Monitoring

Integrate into CI/CD for quality checks:

```python
from experiments.qualitative_analysis import generate_quality_report

report = generate_quality_report(
    original_path="data/raw/openea/BBC_DB",
    augmented_path="results/latest/augmentation",
    output_dir="results/latest/quality"

```

## 🛠️ Configuration

All analyzers use sentence transformers for semantic analysis. Change model:

```python
from experiments.qualitative_analysis import DiversityAnalyzer

analyzer = DiversityAnalyzer(embedding_model="paraphrase-MiniLM-L6-v2")
```

Available models:
- `all-MiniLM-L6-v2` (default) - Fast, good quality
- `paraphrase-MiniLM-L6-v2` - Better for paraphrase detection
- `all-mpnet-base-v2` - Higher quality, slower
